Vestigant Spotlight V1.6.85 implementation notes

Source of truth reviewed first: VestigantSpotlightInv_V1_6_84.zip / ai_context.md.

Primary verified defect from V1.6.84 source:
- GUI AFF4/full workflow set maxNativeRecords=0 but did not set maxNativeRecordsExplicit=true.
- app_runner staged AFF4 parsing defaulted to 25000 unless maxNativeRecordsExplicit was true.
- Result: GUI full/no-guardrails could still hit the 25000 parser cap even though CLI/thin wrapper used MaxNativeRecords=0.

Implemented fixes:
- GUI full/no-guardrails now sets maxNativeRecordsExplicit=true.
- Added guiFullNoGuardrails option/case metadata and effective parser cap logging.
- AFF4 wrappers default to uncapped parser settings for the macOS AFF4 validation workflow.
- Added Cache text full decoded_text to review/search surfaces and global keyword search values.
- Added AFF4/APFS inventory materialization into image_file_inventory.
- Added APFS/image-inventory active comparison branch with lead-only language for missing/not-found candidates.

Local validation:

- Static audit passed.
