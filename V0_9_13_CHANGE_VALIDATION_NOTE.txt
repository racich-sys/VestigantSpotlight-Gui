# V0_9_15 Change and Validation Note

Implemented in V0_9_15:

- Focused the iOS Investigation default view list on Spotlight/CoreSpotlight review and direct FFS overlap. Full FFS/app-database inventory views remain exported/supporting data, but are no longer default iOS interactive review views.
- Reordered the iOS review list so the first/default view is the small Store Parse Summary, followed by Spotlight high-value timeline/entity/date/FFS-overlap views.
- Changed the iOS Investigation tab quick buttons to launch Spotlight-first views: timeline, entities, entity summary, date evidence, FFS overlap, parser targets, file/URL/account references, referenced paths, missing FFS, residency, and store summary.
- Disabled balloon tooltips for the view list; mouse-over now updates the help/summary panel above the table only.
- Prevented the search box from resizing over the Update/Cancel/Rows controls on wide windows.
- Lowered the default interactive page-size box from 1000 to 500 rows to reduce initial load latency. Users can still increase the page size when needed.
- Carried forward reuse-cache development behavior that skips full container hashing by default.
