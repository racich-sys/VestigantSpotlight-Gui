Vestigant Spotlight V0_9_17 change/validation note

Purpose
-------
V0_9_17 addresses the V0_9_16 large-case regression where iOS CoreSpotlight dbStr/property decoding successfully recovered many more values, but normal reused-cache investigator mode still materialized and exported too much low-level native data.

Observed V0_9_16 failure characteristics
----------------------------------------
- SQLite case DB grew to ~36.8 GB.
- SQLite WAL grew to ~31.1 GB.
- ios_string_probe_values exported ~14.49M rows in 58 parts.
- ios_spotlight_communication_candidates exported ~1.44M rows in 6 parts.
- The run was stopped while still exporting ios_spotlight_record_review.

Implemented V0_9_17 changes
----------------------------
1. Default iOS CoreSpotlight key/value persistence is now filtered.
   - Full dbStr/property decoding can still run.
   - Normal iOS investigator mode stores only high-value raw key/value rows by default.
   - Date candidates are still preserved in raw_date_candidates for timeline/date provenance.
   - Full raw native key/value persistence is now opt-in with --diagnostic-full-native-db.

2. Default stored raw values are capped for iOS investigator mode.
   - Oversized text/XML/blob-like values are truncated to prevent pathological database growth.
   - Full values require --diagnostic-full-native-db.

3. Broad support exports are now opt-in.
   - The following are suppressed from normal investigator exports unless support/diagnostics/full export mode is selected:
     spotlight_active_file_comparison.csv
     image_file_inventory.csv
     snapshot_date_warnings.csv
     artifact_dates_wide.csv
     ios_record_investigation_hints.csv
     ios_string_probe_values.csv
     ios_record_string_probe_summary.csv
     ios_timeline_index_updates.csv
     ios_spotlight_communication_candidates.csv
     ios_spotlight_referenced_paths.csv

4. Added CLI flag:
   --diagnostic-full-native-db
   This is intended only for bounded support runs because it can create millions of rows.

Validation performed in sandbox
-------------------------------
- Source updated to 0.9.17/V0_9_17.
- C++ syntax-only checks attempted for modified parser, CLI, app, and exporter files.
- SQL raw-string scan run for nested R"SQL markers and oversized SQL segments.
- Linux CMake build attempted where time permits.
- Full ZIP and patch ZIP integrity checks generated for release package.

User-side validation required
-----------------------------
- Windows/MSVC build.
- CLI version check.
- Self-test.
- V0_9_17 reused-cache run against the same iOS FFS/cache.

Expected V0_9_17 validation signal
----------------------------------
- raw_records should remain about 344,445 for the current test set.
- raw_date_candidates may remain large but should stay defensible as date provenance.
- raw_key_values should be much lower than V0_9_16's 17,347,495 rows.
- SQLite DB and WAL should not expand into tens of GB during normal investigator/reuse-cache mode.
- Normal exports should not include ios_string_probe_values or other massive support-only CSVs unless diagnostic/support mode is explicitly selected.
