Vestigant Spotlight v0.9.17 change/validation note

Primary focus
- Use Apple Core Spotlight developer documentation as a public semantic field taxonomy.
- Use Yogesh Khatri spotlight_parser/mac_apt work as behavioral reference only for iOS CoreSpotlight dbStr map handling.
- Implement clean-room C++ dbStr map loading for iOS CoreSpotlight index.spotlightV2 folders.

Significant changes
1. Native iOS CoreSpotlight dbStr map loading
   - Detects iOS/CoreSpotlight Store-V2 stores where header idx11 is zero.
   - Treats the containing index.spotlightV2 folder as the evidence unit, not store.db alone.
   - Attempts to load:
     dbStr-1.map.data / .offsets / .header as property map
     dbStr-2.map.data / .offsets / .header as category map
     dbStr-4.map.data / .offsets / .header as index map 1
     dbStr-5.map.data / .offsets / .header as index map 2
   - Uses an independent C++ implementation; GPL Python source was not copied.

2. New native dictionary diagnostics
   - native_dbstr_map_inventory
   - native_category_dictionary
   - native_index_dictionary_summary
   - existing native_property_dictionary now receives iOS dbStr-derived property rows when available.

3. New iOS Spotlight review views/exports
   - iOS - Spotlight dbStr Map Inventory
     exports/ios_spotlight_dbstr_map_inventory.csv
   - iOS - Spotlight Dictionary Coverage
     exports/ios_spotlight_dictionary_coverage.csv
   - iOS - Spotlight Apple Field Coverage
     exports/ios_spotlight_apple_field_coverage.csv
   - Support exports:
     native_dbstr_map_inventory.csv
     native_category_dictionary.csv
     native_index_dictionary_summary.csv

4. GUI additions
   - Added dbStr/dictionary/Apple-field coverage views to the iOS Investigation default Spotlight-first view list.
   - Help text explains that missing dbStr components usually explain generic probe-only decoding.

5. Thin upload samples
   - Added sampled upload CSVs for dbStr map inventory, dictionary coverage, and Apple field coverage.
